# Contributing to the Vancouver Airbnb Analyzer:

Thank you for your interest in contributing to the **Vancouver Airbnb Analyzer** dashboard!  
This document outlines how you can contribute effectively and respectfully.

---

## How to Contribute:  
  
There are many ways to support this project:

- Reporting bugs or unexpected behavior
- Suggesting new features, visualizations, or analytical insights
- Improving docs and 
- Enhancing code structure, performance, or design
- Improving dashboard visuals or ui functionalities 
- Improving models and analytics methods

All forms of contributions are appreciated!

---

## Identifying Issues:

If you identify a problem, please open a GitHub Issue and include:

- A concise and descriptive title
- Clear steps to reproduce the issue (if applicable)
- Expected behavior vs. observed behavior
- Screenshots or error messages when relevant
- An appropriate and unique title identifying the issue

Providing detailed steps on how to reproduce the issue in order to help developers pinpoint the cause.

---

## Improvement Suggestions:

We encourage thoughtful suggestions that improve the dashboard’s usability, interpretability, or impact.

When 

- The motivation behind your suggestion
- How it improves analysis or user experience
- Any foreseeable challenges or implementation considerations
- Consideration of how these suggestions could affect other areas of the project

---

## Implementing Contributions:

We follow a pull request–based workflow.

Before opening a pull request:

- Ensure your changes are focused and well-documented
- Sync with the latest version of the main branch
- Maintain clean, readable, and structured code
- Include meaningful commit messages
- Test your changes where applicable

Be open to feedback and collaborative refinement during the review process.

---

## Development Setup  
  
Work on your current branch from the main branch relating to your current contribution or issue. Create a pull request to merge into the main branch. Pull requests require at least one reivew approval before merging.

---

## Collaboration Standards

We are committed to maintaining a respectful, inclusive, and constructive environment.

All contributors are expected to:

- Communicate professionally
- Provide constructive feedback
- Respect diverse perspectives
- Support a collaborative learning experience

---

## Questions or Support

If you need clarification or guidance:

- Open a GitHub issue and label it appropriately
- Use the repository discussion section for general questions
- Contact gnandtandre@outlook.com for further questions

We appreciate your time, effort, and thoughtful contributions to this project.
